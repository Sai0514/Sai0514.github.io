Zepto击穿问题

击穿现象：
  1. 同页面tap点击弹出弹层，弹层中也有一个button，正好重叠的时候，会出现击穿 tap事件点击。
  2. 当前页面tap点击页面跳转，新页面中同位置也有一个按钮，会出现击穿。
击穿原因：

Zepto的插件touch.js中的tap事件，可以解决移动浏览器中300ms延迟的问题。但是会导致击穿现象。

click 事件有延迟（大概是300ms，为了实现safari的双击事件），所以在执行完 tap 事件之后，弹出层立即隐藏，此时 click 事件还在延迟的 300ms 之中。当 300ms 到来的时候，click 点击到的其实是隐藏元素下方的元素。

如果正下方的元素有绑定 click 事件，此时便会触发，如果没有绑定 click 事件的话就不发生。如果正下方的是 input 输入框（或 select / radio / checkbox），点击默认 focus 而弹出输入键盘，也会出现击穿现象。

解决办法：

  1. 使用fastclick.js解决移动端浏览器300ms问题；

    fastclick.js实现思路是，取消click事件，用touchend模拟快速点击行为
    FastClick.attach(document.body)

  2. 遮挡

    click事件的滞后性导致的击穿问题。因此可以给元素的消失做一个fade效果，类似jQuery里的fadeOut，并设置动画duration大于300ms，这样当延迟的click触发时，就不会“穿透”到下方的元素了。
    setTimeout(function(){},350)

  3. pointer-events

    pointer-events是CSS3中的属性，它有很多取值，有用的主要是auto和none。
    这种方案没有在项目中用过，下次可以尝试一下。

网站的术语总结

在百度实习期间，接触到了网页统计的一些术语，作为埋点统计的一种方法，例如DP统计，DP监控。如下总结了一部分术语，作为简单的总结。

1. DP初始化配置
2. DP监控模块
3. PV、UV、IP
PV

PV(page view): 页面浏览量或者点击量，是衡量一个网站或网页用户访问量。PV值就是所有访问者在24小时(0点到24点)内看了某个网站多少个页面或某个网页多少次。PV即页面刷新的次数，每一次页面刷新，就算做一个次PV流量。

度量方法: 从浏览器发出一个网络服务器的请求(Request)，网络服务器接到这个请求之后，会将该请求对应的一个网页(Page)发送到浏览器，从而产生了一个PV。因此，只要是这个请求发送给了浏览器，无论这个页面是否完全打卡(下载完成)，那么都是应当计为1个PV。

UV

UV（unique visitor）即独立访客数，指访问某个站点或点击某个网页的不同IP地址的人数。在同一天内，UV只记录第一次进入网站的具有独立IP的访问者，在同一天内再次访问该网站则不计数。

UV提供了一定时间内不同观众数量的统计指标，而没有反应出网站的全面活动。通过IP和cookie是判断UV值的两种方式。

IP

IP可以理解为独立IP的访问用户，指1天内使用不同IP地址的用户访问网站的数量，同一IP无论访问了几个页面，独立IP数均为1。但是假如说两台机器访问而使用的是同一个IP，那么只能算是一个IP的访问。

Git

  1. 常用命令

    • git status // 查看分支状态，查看文件状态
    • git add . // 把要提交的所有修改放到暂存区（Stage）
    • git commit -m ‘add new info’ //可以一次性把暂存区的所有修改提交到分支
    • git pull –rebase 分支 // 解决冲突的过程

    • git push origin [自己的分支名称]:refs/for/[自己的分支名称]%r=评审人邮

    • git checkout -b dev // 创建dev分支，然后切换到dev分支

      等价于
      git branch dev //创建分支

    • git checkout dev //切换分支
    • git branch //查看分支状态
    • git log // 查看历史记录
    • git merge dev 把dev分支的工作成果合并到master分支上

  2. 遇到过的问题：

    子目录出现 .git 影响整个项目的push。

    (modified content, untracked content)

  3. 解决办法：

    • cd new directories/sub directories (切换至新加目录或子目录)
    • rm -rf .git (删除新加目录或子目录下的.git)
    • cd main direactory (返回到项目主目录)
    • git rm -rf –cached new directories/sub directories (删除缓存)
    • git add . (重新add，commit，pull，push)
