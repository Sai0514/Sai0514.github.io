git && git pull --rebase

2018-05-30

1、 遇到的问题：

modified: xxx(modified content, untracked content)

意思是：目录没有被跟踪

解决办法：

  1. cd new-directories/sub-directories (切换至新加目录或子目录)
  2. rm -rf .git (删除新加目录或子目录下的.git)
  3. cd main-direactory (返回到项目主目录)
  4. git rm -rf –cached new-directories/sub-directories (删除缓存)
  5. git add . (重新add，commit，pull，push)

2、 git pull –rebase

开发过程中，大家一般都会基于dev分支并行开发。然后在开发过程中，需要经常拉取dev分支上的代码以便及时更新。
—- rebase 并不会产生一个commit提交，而是会将你的commit附加到上一个commit的结尾处。在看commit log时，就不会多出你所不知道的commit出来。因此，git pull –rebase可以让你的代码提交看起来更加的流畅干净。
