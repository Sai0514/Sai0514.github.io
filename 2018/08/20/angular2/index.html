angular2

2018-08-20

angular2 其实是比较老的框架了。这次写需求，也只是个学了皮毛，希望后期能把rxjs也用上。

  1. 首先写的是新组件，我们创建组件 ng generate component myapp
  2. 创建完之后，新组件目录下有三个文件 myapp.component.html, myapp.component.css, myapp.component.ts(新版本脚手架 angular-cli 应该还会生成其他文件 myapp.module.ts、myapp.component.spec.ts，主要是为了更好地管理代码.)

  3. 对于component组件，这里有的说：

    • 首先，项目都有一个根组件app，如果我们的组件需要在app根组件引用，那么必须在app.module.ts 中引入。本项目为叙述简单，直接采用根组件作为父组件。
    • 其次，我们介绍 myapp 组件，这个组件我主要是请求接口，访问并显示数据，生成定制化的业务组件。

    • 接下来，我们详细说 myapp 组件：

      1. 第一，我们需要做引入： 在父组件(此处是根组件app) app.component.ts import { MyAppComponent } from './myapp/myapp.component'；，在 app.module.ts 引入模块

      2. 第二，我们需要做输出： 在子组件(此处是业务组件myapp) myapp.component.ts export class MyAppComponent { /* 此处可以做数据传输 */};，在 myapp.module.ts 模块引入代码

      3. 第三，我们需要做处理： 例如， 我这个组件是需要父子组件的传递的。首先，我开放了两个接口，也是两类方法。

        (一) 父组件传递给子组件数据，在子组件显示。这个比较简单，在父组件 app.component.html 加入子组件<my-app [child_data]='parent_data'></my-app>在子组件 myapp.component.ts 的构造函数之前 @Input child_data: child_data 引入传递的数据；在 ngOnChanges 生命周期钩子函数里面可以监测出这个数据的变化。
        (二) 子组件给父组件传递数据，在父组件 app.component.ts 中引入子组件，并且@ViewChild(MyAppComponent) MyAppComponent, 之后可以在 ngOnInit 生命周期钩子函数里初始化子组件的方法。

    • 最后，我们再说说请求接口： angular2 对于接口服务，单独拿出来做处理，在 app.component.spec.ts 中写好接口，并对外抛出来。 在 app.component.ts 中引入 service 服务，在构造函数 contructor(/* 注册服务，声明service */)，这样便可以引用 service 里面的接口方法了。
