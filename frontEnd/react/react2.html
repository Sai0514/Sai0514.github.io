React之进阶学习

1. 状态提升

将组件之间共享的状态交给组件最近的公共父节点保管，然后通过props把状态传给子组件，这样就可以实现在组件之间共享数据了。

例如： 子组件Input中输入框输入数字，同时将改变的值传入父组件App，父组件再将状态值通过props传给子组件，由子组件将值实时显示出来。难点：主要在于子组件Input中的改变值如何传给父组件，其实是通过事件绑定的方法传过去的。

import React, { Component } from 'react';
import ReactDOM from 'react-dom';
import './index.css';
import registerServiceWorker from './registerServiceWorker';
class Input extends Component {
     handleChangeNum(e){
          if (this.props.onSubmit) {
              const num  = e.target.value
              this.props.onSubmit(num)
        }
      }
      render () {
        return (
              <div>
                <input type='number' onChange={this.handleChangeNum.bind(this)}/>
              </div>
        )
      }
}
class PercentageShower extends Component {
      static defaultProps = {
          num: 0
      }
      render () {
          const number = ((this.props.num)*100).toFixed(2)
        return (
              <div>{number}%</div>
        )
      }
}
class PercentageApp extends Component { 
      constructor(props) {
          super(props);
          this.state = {
              num: 0
          }
      }
      handleSubmitComment (num) {
        // console.log(num)
        this.setState({
              num: num
        })
      }
      render () {
        return (
          <div>
              <Input onSubmit={this.handleSubmitComment.bind(this)}/>
              <PercentageShower num={this.state.num}/>
          </div>
        )
      }
}
ReactDOM.render(<PercentageApp />, document.getElementById('root'));

2. 组件生命周期

(1) React.js将组件渲染，并且构造dom元素然后塞入页面的过程称为组件的挂载。这一节我们学习了React.js控制组件在页面上挂载和删除过程里面的几个方法：

  • constructor()
  • componentWillMount() // 挂载元素开始之前，即组件调用render方法之前
  • render()
  • componentDidMount() // 组件挂载完成之后，即DOM元素已经插入页面后调用
  • …
  • componentWillUnmount() // 组件对应的DOM元素从页面中删除之前调用

(2) 组件 constructor()、componentWillMount()、componentDidMount() 、componentWillUnmount()的具体过程及作用：

  • constructor() // 初始化状态：state
  • componentWillMount() // 组件启动：ajax数据拉取、定时器启动等
  • render()
  • componentDidMount() // 组件加载完成后的启动工作：例如：动画的启动
  • …
  • componentWillUnmount() // 组件销毁：数据的清理、定时器的清理

例如：

class Post extends Component {
      constructor(){
        super()
        this.state = {
              postContent: ''
        }
      }
      componentWillMount() {
        this._loadData()
      }
      async _loadData(){
        this.setState({ postContent: '数据加载中...'})
        const content = await getPostData()
        this.setState({postContent: content})
      }
      render () {
        return (
             <div>
                <div className='post-content'>{this.state.postContent}</div>
                <button onClick={()=>{this._loadData()}}>刷新</button>
              </div>
        )
      }
}

(3) 补充关于更新阶段的组件生命周期：

  • shouldComponentUpdate(nextProps, nextState)：可以通过这个方法控制组件是否重新渲染。如果返回 false 组件就不会重新渲染。这个生命周期在 React.js 性能优化上非常有用。
  • componentWillReceiveProps(nextProps)：组件从父组件接收到新的 props 之前调用。
  • componentWillUpdate()：组件开始重新渲染之前调用。
  • componentDidUpdate()：组件重新渲染并且把更改变更到真实的 DOM 以后调用

3. React中的DOM操作

  • ref

React.js提供了on*帮助进行事件监听，所以不必直接调用addEventListener的DOM API； React.js 直接通过 setState方式重新渲染组件，渲染的时候将新的props传递给子组件；React.js 还需要提供ref属性帮忙获取已经挂载的元素dom节点，可以给某个jsx元素加上ref属性。

  • props.children

如果组件标签也能像普通的 HTML 标签那样编写内嵌的结构，那么就方便很多了。实际上，React.js 默认就支持这种写法，所有嵌套在组件中的 JSX 结构都可以在组件内部通过 props.children 获取到。开发中：使用自定义组件的时候，可以在其中嵌套 JSX 结构。嵌套的结构在组件内部都可以通过 props.children 获取到

  • style

在 React.js 中你需要把 CSS 属性变成一个对象再传给元素,style 接受一个对象，这个对象里面是这个元素的 CSS 属性键值对，原来 CSS 属性中带 - 的元素都必须要去掉 - 换成驼峰命名.

const getDefaultStyledPost = (defaultStyle) => {
      return (
        class Post extends React.Component {
              render() {
                const newStyle = {...defaultStyle, ...this.props.style}
                return (
                     <p style={newStyle}>段落</p>
                )
              }
        }
      )
}

出于安全考虑的原因（XSS 攻击），在 React.js 当中所有的表达式插入的内容都会被自动转义，就相当于 jQuery 里面的 text(…) 函数一样，任何的 HTML 格式都会被转义掉。需要给 dangerouslySetInnerHTML 传入一个对象，这个对象的 __html 属性值就相当于元素的 innerHTML，这样我们就可以动态渲染元素的 innerHTML 结构了。
