React之初级认识

1. JSX语法

JSX其实就是JavaScript对象。
首先：如何用Javascript对象来表现一个DOM元素的结构。
HTML结构：

<div class='box' id='content'>
       <div class='title'>Hello</div>
      <button>Click</button>
</div>

JavaScript对象：

{
  tag: 'div',
  attrs: { className: 'box', id: 'content'},
  children: [
    {
      tag: 'div',
      arrts: { className: 'title' },
      children: ['Hello']
    },
    {
      tag: 'button',
      attrs: null,
      children: ['Click']
    }
  ]
}

因此，DOM 元素包含的信息其实只有三个：标签名，属性，子元素。React.js 就把 JavaScript 的语法扩展了一下，让 JavaScript 语言能够支持这种直接在 JavaScript 代码里面编写类似 HTML 标签结构的语法，即JSX语法。

总结：

  • JSX 是 JavaScript 语言的一种语法扩展，长得像 HTML，但并不是 HTML。
  • React.js 可以用 JSX 来描述你的组件长什么样的。
  • JSX 在编译的时候会变成相应的 JavaScript 对象描述。
  • react-dom 负责把这个用来描述 UI 信息的 JavaScript 对象变成 DOM 元素，并且渲染到页面上。

2. render()方法

一个组件类必须要实现一个 render 方法，这个 render 方法必须要返回一个 JSX 元素。但这里要注意的是，必须要用一个外层的 JSX 元素把所有内容包裹起来。返回并列多个 JSX 元素是不合法的。例如：

render () {
      return (
        <div>  // 当有并列JSX元素时，需要外层div包裹
              <div>第一个</div>
              <div>第二个</div>
        </div>
      )
}

重点： {} 内可以放任何 JavaScript 的代码，包括变量、表达式计算、函数执行等等。render 会把这些代码返回的内容如实地渲染到页面上。

3. 事件监听on*

React封装了不同类型的事件，这些on*事件监听只能在普通HTML标签上，不能用在组件标签上，即<Header onClick={…} />是不可以的。和普通浏览器一样，事件监听函数会自动传入一个event对象，这个对象和普通浏览器event所包含的方法和属性基本一致。例如：

class Title extends Component {
      handleClickOnTitle (e) {
        console.log(e.target.innerHTML)
      }
      render () {
        return (
              <h1 onClick={this.handleClickOnTitle}>React 小书</h1>
        )
      }
}

React.js 的事件监听方法需要手动 bind 到当前实例。例如：

class Footer extends Component {
      handClickFooter(word, e) {
          console.log(word)
      }
      render () {
        return (
            <div>
                  <h2 onClick={this.handClickFooter.bind(this, 'this is liruonan')}>This is footer</h2>
            </div>
        )
      }
}

总结:

  • 一般某个类的实例方法里面的this指的是这个实例本身，但是react调用传给它的方法时，并不是通过对象方法的方式调用，而是直接通过函数调用，因此事件监听函数内并不能通过this获取到实例，此时需要将实例方法bind到当前实例上再传入给react。bind会将实例方法绑定到当前实例上，然后再把绑定后的函数传给React.js的onClick事件监听。
  • 这种 bind 模式在 React.js 的事件监听当中非常常见，bind 不仅可以帮我们把事件监听方法中的 this 绑定到当前组件实例上；还可以帮助我们在在渲染列表元素的时候，把列表元素传入事件监听函数当中

4. 组件状态state

React中使用state存储可变化的状态。setState()方法由父类Component所提供。当调用该函数时，React.js会更新组件的状态state，并且重新调用render()方法，然后再把render()方法所渲染的最新内容显示到页面上。
setState传入参数

constructor (props) {
       super(props)
       this.state = {
          name: 'Tomy',
          isLiked: false
    }
  }
  handleClickOnLikeButton () {
    this.setState({
          isLiked: !this.state.isLiked
    })
  }

setState接受函数参数

handleClickOnLikeButton () {
  this.setState((prevState) => {
        return {count: 0}
  })
  this.setState((prevState) => {
      return {count: prevState.count + 1}
  })
  this.setState((prevState) => {
      return {count: prevState.count + 2}
  })
}

总结：

  • 当我们改变组件的状态时，不能直接使用this.state = xxx 这种方式来修改，如果这样做了，React就无法知道你修改了组件的状态，也就没办法渲染新页面。因此，一定要使用setState方法，它接受一个对象或者函数作为参数。
  • 当调用setState的时候，React.js并不会马上修改state，而是把这个对象放到一个更新队列里面，稍后才会从队列中把新状态提取出来合并到state中，然后再触发组件更新。【状态缓存】

5. 可配置props

如何让组件能适应不同场景下的需求，我们就要让组件具有一定的“可配置”性。React中的props可以帮助我们达到这个效果，每个组件都可以接受一个props参数，其可以是一个对象，包含所有对这个组件的配置。例如：

class LikeButton extends Component {
      constructor () {
        super()
        this.state = { isLiked: false }
      }
      handleClickOnLikeButton () {
        this.setState({
              isLiked: !this.state.isLiked
        })
      }
    render () {
        const likedText = this.props.likedText || '取消'
        const unlikedText = this.props.unlikedText || '点赞'
        return (
              <button onClick={this.handleClickOnLikeButton.bind(this)}>
    {this.state.isLiked ? likedText : unlikedText} 👍
              </button>
        )
      }
}
class Index extends Component {
      render () {
        return (
              <div>
                <LikeButton likedText='已赞' unlikedText='赞' />
              </div>
        )
      }
}

通过把参数放在标签的属性中，所有属性都会作为props对象的键值传入给子组件。从render()函数可以看出，组件内部通过this.props的方式获取到组件的参数。如果没有相应的属性，可使用默认属性static defaultProps={ }

6. state vs props

  1. state的主要作用是组件保存、控制、修改自己的可变状态。state在组件内部初始化，可以被组件自身修改，而外部不能访问也不能修改。因此，state是一个局部的、只能被组件自身控制的数据源。state中状态可以通过this.setState方法更新，但会导致重新渲染。
  2. props的主要作用是让使用该组件的父组件可以传入参数来配置该组件。它是外部传进来的配置参数，组件内容无法控制也无法修改。除非外部组件主动传入新的props。
  3. 两者的关系：都可以决定组件的行为和状态。一个组件的state中数据可以通过props传给子组件，一个组件可以使用外部传入的props来初始化自己的state。但是职责分明：state是让组件控制的状态，props是让外部对组件自己配置。
  4. 总之： 少用state，多用props
