Day1

  1. 盒子模型

    网页中元素占有空间的大小由4个部分组成，包括元素的内容content,元素的内边框padding,元素的边框border，元素的外边距margin。这四个部分，content表示显示的内容，padding表示填充区域，border是边框，margin用来分隔相邻区域。

  2. href和src

    href指向网络资源所在位置，建立和当前元素或当前文档之间的链接，用于超链接。
    src指向外部资源位置，指向的内容将会嵌入到文档中当前标签所在位置。在请求src资源会将其加载、编译、执行处理，此时会停止其他资源的加载，因此js文件一般放在底部。

  3. 同步和异步

    同步是阻塞模式，异步是非阻塞模式

    同步指一个进程在执行某个请求的时候，若该请求需要一段时间才能返回信息。那么该进程会一直等待下去，直到收到返回信息才能继续执行下去；

    异步指进程不需要一直等下去，而且继续执行下面的操作，不管其他进程的状态。当有消息返回时系统会通知进程进行处理，提高执行效率。

  4. 怎样添加、移除、移动、复制、创建和查找节点？

    创建新节点

    • createDocumentFragment() //创建一个DOM片段
    • createElement() //创建一个具体元素

    • createTextNode() //创建一个文本节点

      添加、移除、替换、插入节点

    • appendChild()
    • removeChild()
    • replaceChild()

    • insertBefore()

      查找节点

    • getElementByTagName() //通过标签名称
    • getElementByName() //通过元素Name属性值
    • getElementById() //通过id，唯一性

Day2

  1. 一次完整的HTTP事务过程

    基本流程：

    • a. 域名解析
    • b. 发起TCP三次握手
    • c. 建立TCP连接后发起http请求
    • d. 服务器端响应http请求，浏览器得到html代码
    • e. 浏览器解析html代码，并请求html代码中的资源
    • f. 浏览器对页面进行渲染，呈现给用户

  2. web攻击技术

    • (XSS) 跨站脚本攻击： 通过存在安全漏洞的Web网站注册用户的浏览器内运行非法的html标签或js进行的一种攻击。如： 盗取用户的cookie，改变网页的DOM结构，重定向到其他网页等。
    • (CRSF)跨站点请求伪造： 指攻击者通过设置好的陷阱，强制对已完成的认证用户进行非预期的个人信息或设定信息等某些状态更新。如：攻击者盗用了你的名义，以你的名义发送了请求。
    • (SQL) SQL注入攻击： 通过在对SQL语句调用方式上的疏漏，恶意注入SQL语句。

  3. ajax

    ajax请求：

    • 同步异步模式，提升了用户体验
    • 优化了浏览器和服务器之间的传输，减少不必要的数据往返，减少了带宽占用

    • ajax在客户端运行，承担了一部分由服务器承担的工作，减少了大用户量下的服务器负载

      ajax特点：

    • 实现动态不刷新(局部刷新)

    • readyState属性： 状态有5个可取值： 0 = 未初始化； 1 = 启动； 2 = 发送； 3 = 接收； 4 = 完成

      ajax缺点：

    • 不支持浏览器back按钮

    • 安全问题ajax暴露了与服务器交互的细节
    • 对搜索引擎的支持比较弱
    • 破坏了程序的异常机制
    • 不容易调试

  4. 优雅降级和渐进增强

    渐进增强：

    针对低版本浏览器进行构建页面，保证最基本的功能，然后再针对高级浏览器进行效果、交互等改进和追加功能达到更好的用户体验。

    优雅降级：

    一开始就构建完整的功能，然后再针对低版本浏览器进行兼容。

  5. 前端网页制作怎么克服不同分辨率的问题

    根据屏幕不同大小，缩小窗口处横向滚动条在所难免，但理想情况下，页面应该能适应不同客户端浏览器和分辨率。实际操作通常又有三种情况： 版面自适应、视觉自适应、内容自适应。

  6. 是否该继续使用<b> 和 <i> 标签

    我认为字体的样式都应该用css来控制，不应该使用标签来控制。所以我依然会采用<strong>和 <em>标签，如果是大量的文件，那么可以直接使用font-weight和font-style来解决。个人不建议使用<b>和<i>标签，但也会尊重个人习惯。

Day3

  1. 什么是伪数组？将伪数组更改为标准数组？

    伪数组：无法调用数组方法或期望length属性有特殊行为，但仍可以对真正数组遍历方法来遍历它们。典型的是arguments参数，还有document.getElementByTagName, document.childNodes之类的，它们都返回NodeList对象。

    转化成标准数组：

     // 使用ES5 
 Array.prototype.slice.call(fakeArray)
 Array.prototype.slice.apply(this,fakeArray)
 // 使用ES6 
 Array.from(fakeArray)

  2. 浏览器本地存储

    html5中的Web Storage包括两种存储方式：sessionStorage和localStorage。

    sessionStorage：用于本地存储一个会话session中的数据，这些数据只有在同一个会话中的页面才能访问并且当会话结束后数据也随之销毁。会话级存储。

    localStorage：用于持久化的本地存储，除非主动删除数据，否则数据是永远不会过期的。持久化存储。

  3. 线程与进程的区别

    进程和线程都是一个时间段的描述，是CPU工作时间段的描述，不过是颗粒大小不同。进程是应用程序的执行实例。由于一个进程可以由多个线程，线程可以被认为是“轻量级”的过程。

    一个程序至少有一个进程，一个进程至少有一个线程。进程是cpu资源分配的最小单位，线程是cpu调度的最小单位。以前进程既是资源分配也是调度的最小单位，后来为了更合理的使用cpu(实际上是cpu性能越来越好)，才将资源分配和调度分开，就有了线程。

  4. 减少页面加载时间的方法

    • 优化图片
    • 图像格式的选择
    • 优化CSS(压缩合并CSS)
    • 网址后加斜杠
    • 标明高度和宽度(如果没有，浏览器需要边下载边计算，不断调整页面，不但影响速度，也影响体验)
    • 减少http请求(合并文件，合并图片)

  5. 在链接中应该使用诸如 ‘click here’ 一类的笼统词汇

    为了提高站点的易用性、可访问性和SEO性能，应该始终使用描述性链接。用户在点击之前，只能通过周围的上下文关系猜测这个链接是做什么的。我不认为’Click here’可以获得更多点击率。

  6. 链接是否应该在新窗口打开

    总体来讲，应该避免使用新窗口打开链接，但是在部分场合，如帮助链接等可以使用新窗口。为了提高易用性，最好在需要打开新窗口的地方，用小图标提示一下。
