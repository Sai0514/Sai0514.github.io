一、基础知识

Node三要素： 异步I/O、事件驱动、单线程

1、 异步I/O

  • 异步IO操作，在前端是一个常见的场景。

例如：发起ajax请求，我们知道回调函数会在异步请求结束之后执行，但并不知道具体的时间点。

  • 在Node中，大部分操作是异步的。因此，node底层构建了很多异步IO的API。

例如：从文件读取到网络请求等。

2、 事件与回调函数

-Node将前端浏览器中的很多事件引入后端，配合异步I/O，将事件点暴露给业务逻辑。

例如：Node创建一个Web服务器，并侦听8080端口。对于服务器，为其绑定request事件；对于request请求对象，为其绑定data事件和end事件。

3、 单线程

Node保持了javascript在浏览器中单线程的特点。

Node采用了与Web Workers相同的思路来解决单线程中的大计算量问题：child_process.
原理：Web Workers —

4、 应用场景

I/O密集型：node利用事件循环的处理能力，而不是启动每个线程为每一个请求服务。

三、Node模块实现

1、路径分析

浏览器会对引入的模块进行缓存，以减少二次引入时的开销。

  • 模块标识符
    • 核心模块，如：http、fs、path等
    • . 或 .. 开始的相对路径文件模块
    • 以 / 开始的绝对路径文件模块
    • 非路径形式的文件模块，如自定义的connect模块
    • 核心模块的优先级仅次于缓存加载，因为其在Node的源代码中已经编译为二进制文件。
    • . 或 .. 开始的相对路径文件模块和 - 以 / 开始的绝对路径文件模块.require将其转化为真实路径，并以真实路径为索引，将编译执行后的结果存放到缓存中。
    • 自定义模块，需要查询模块路径，逐个尝试路径地址，直到找到目标文件。

2、文件定位

  • 文件扩展名分析

    require()分析标识符，会按照.js、.json、.node顺序补足扩展名依次尝试，建议.node和.json带上扩展名。

  • 目录分析和包

    Node在当前目录下查找package.json文件，通过JSON.parse()解析出包描述，取出main属性指定的文件名进行定位。如果缺少扩展名，则进入扩展名分析的步骤。

    3、编译执行

    模块编译一般指文件模块，即用户自己编写的模块。

  • .js文件的模块编译

    1 Node对获取的js文件进行包装，(function (exports, require, module, _filename, _dirname){}

    (function (exports, require, module, _filename, _dirname){

    // 例如： math模块
var math = require('math');
exports.area = function(radius){
    return Math.PI * radius * radius;
};

    });
    >

    2 包装之后的代码通过vm原生模块的runInThisContext()方法执行，返回一个具体的function对象。

    3 执行之后，模块的exports属性被返回给了调用方，exports属性上的任何属性和方法都可以被外部调用。

  • .node文件的模块编译

    Node调用process.dlopen()方法进行加载和执行，通过中间层libuv兼容跨平台。
    .node模块文件是C/C++模块扩展编译而来，只需要加载和执行。

  • .json文件的模块编译

    Node利用fs模块同步读取json文件，调用JSON.parse()方法得到对象，然后将它赋给模块对象的exports。
