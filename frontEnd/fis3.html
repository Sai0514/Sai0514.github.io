FIS3操作

1.安装

先安装nodejs(有版本要求version<7)，再使用npm install -g fis3 全局安装

2.配置

match

fis.match(selector, props); // 配置接口，匹配css，js，less，png等资源，通过pros配置规则。配置规则如： useHash:true; release:’/dist/$0’; 等

  • fis.match(‘*.{js,css,png}’, { useHash: true }); // 文件指纹，加md5戳
  • fis.match(‘*.js’,{ optimizer: fis.plugin(‘uglify-js’) }); // js压缩
  • fis.match(‘*.css’, { optimizer: fis.plugin(‘clean-css’) }); // css压缩
  • fis.match(‘*.png’, { optimizer: fis.plugin(‘png-compressor’) }); // png压缩
  • fis.match(‘::package’, { spriter: fis.plugin(‘csssprites’) }); // 启用 fis-spriter-csssprites 插件
  • fis.match(‘*.css’, { useSprite: true }); // 对CSS进行图片合并

release

fis3 release -d // 构建发布到指定目录

  • fis3 release -d ./output // 构建发布到项目目录的output目录
  • fis3 release -d ../dist // 构建发布到项目父级目录下的dist子目录
  • fis3 release // 构建发布到fis内置server下
  • fis3 release -debug -wL // 构建发布到debug环境下，并实现监听和自动刷新

server

fis3 server -h // 启动参数

  • fis3 server open // 打开内置server根目录
  • fis3 server start // 启动内置server
  • fis3 server start – type node // 使用后端模板node，启动内置server
  • fis3 server start – type smarty // 使用后端模板smarty，启动内置server

FIS3应用

.1. 安装插件

npm install -g 插件名

npm install -g fis-parser-less  // 引入less预处理插件
npm install -g fis3-postpackager-loader // 散列文件合并

.2. 预处理

parser: fis.plugin(插件名) 

fis.match('*.less', {
  parser: fis.plugin('less'),  // fis-parser-less插件进行解析
  rExt: '.css'  // .less文件后缀构建后被改成.css文件
})

.3. 简单合并

基于整个项目打包 

fis.match('::package', {
  postpackager: fis.plugin('loader')
})；
fis.match('*.{less,css}', {  // .less先预处理再合并
    packTo: '/static/aio.css'
});
fis.match('*.js', {
    packTo: '/static/aio.js'
});

基于页面的打包

fis.match('::package', {
  postpackager: fis.plugin('loader', {
    allInOne: true
  })
});

.4. 构建调试

进入 demo 目录，执行命令，构建即发布到本地测试服务根目录下：
fis3 release // 按需发布到指定环境

fis3 server start –type smarty // 启动内置server预览，考虑后端模板

fis3 release debug -wc // 发布到调试环境，-wc实时监听
