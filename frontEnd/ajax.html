异步请求之总结

1. 当有两个异步请求发过来的时候，如何判断两个请求都成功？

方法一：

传统解法，定义一个标志位 flag

const AJAX_NUM = 2;
$.ajax({},function A(){
    if (success) AJAX_NUM --;
    if (AJAX_NUM === 0) {renderPage();} 
})
$.ajax({},function B(){
    if (success) AJAX_NUM --;
    if (AJAX_NUM === 0) {renderPage();}
})

方法二：

$.ajax()操作完成后，如果使用的是低于1.5.0版本的jQuery，返回的是XHR对象，你没法进行链式操作；如果高于1.5.0版本，返回的是deferred对象，可以进行链式操作。

(1) 指定同一操作的多个回调函数
deferred对象的一大好处，就是它允许你自由添加多个回调函数。done
表示传统的XHR对象里面的success，fail表示error。

$.ajax("test.html")
　　.done(function(){ alert("哈哈，成功了！");} )
　　.fail(function(){ alert("出错啦！"); } )
　　.done(function(){ alert("第二个回调函数！");} );

(2) deferred对象的另一大好处，就是它允许你为多个事件指定一个回调函数，这是传统写法做不到的。$.when

$.when($.ajax("test1.html"), $.ajax("test2.html"))
　　.done(function(){ alert("哈哈，成功了！"); })
　　.fail(function(){ alert("出错啦！"); });

综上，分析完毕，我们可以知道，使用$.when可以实现多个事件实现同一个回调函数。如上题：

$.when($.ajax(),$ajax()).done(renderPage())

2. 页面链接跳转URL不记录处理

当执行Ajax操作的时候，会往浏览器history中塞入一个地址(使用pushState)这是无刷新的；

浏览器的前进与后退，会触发window.onpopstate事件，通过绑定popstate事件，就可以根据当前URL地址中的查询内容让对应的菜单执行ajax载入，实现ajax前进与后退效果；

页面首次载入的时候，如果没有查询到信息或信息不匹配，则使用第一个菜单的ajax地址的查询内容，并使用history.replaceState更改当前的浏览器历史，然后触发ajax请求。

我们的策略是：当页面加载的时候，使用history.replaceState替换当前的状态，保证前进和后退不保存历史记录。其中，history.replaceState({},title,url)参数url带入时间戳来表示唯一性。

history.replaceState(null, document.title, prefix + 'nocache=' + time);
